# Экзаменационная система

Банки здесь — инструмент экзаменатора, а не конспект для предварительного чтения. В разговоре с Мишей экзаменатор берёт один вопрос за сообщение, ждёт самостоятельный ответ и только затем задаёт follow-up или предлагает короткую практическую проверку. Ключи находятся в HTML-комментариях «ТОЛЬКО ЭКЗАМЕНАТОРУ»: они не выводятся в learner flow и не цитируются до попытки.

## Ритм

- После темы: 3–5 вопросов из её банка.
- После 3–4 тем: практический checkpoint с запуском кода, SQL или конфигурации.
- После модуля: накопительная проверка, где не менее 30% — старые темы.
- На каждой проверке: один неожиданный вопрос из уже изученного (`surprise-old`).
- Слабый ответ не закрывает тему: зафиксировать модель и вернуть её в очередь повторения через 1, 7 и 21 день.

Теги повторения: `R1` — на следующей сессии, `R7` — через неделю, `R21` — через три недели, `surprise-old` — случайный старый вопрос. Вопрос можно считать закрытым только с доказательством, а не с красивой формулировкой.
