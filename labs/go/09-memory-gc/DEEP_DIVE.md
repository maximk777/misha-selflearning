# Deep dive: почему escape — не приговор

> Уровень: `advanced`
> Время: 30 минут
> Не блокирует minimum path.

Выполни `go build -gcflags=-m . 2>&1 | sed -n '1,80p'`. Сравни сообщения компилятора с реальным `go run .`: одно говорит о размещении/времени жизни, другое даёт только грубый снимок памяти процесса.

**Один вопрос:** почему нельзя объявить каждую escape-строку bug без pprof и workload? Ответ: compiler выбирает корректность и может менять стратегию между версиями; сначала измеряем полезную метрику.
