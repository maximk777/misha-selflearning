# Урок
Прочитай `progress/PROFILE.md`, `progress/STATUS.md`, `progress/EVIDENCE.md`, `progress/EXAMS.md`, `progress/REVIEW_QUEUE.md`, `roadmap/SYLLABUS.md` и файл темы.

Веди урок слоями: быстрый результат → реальные backend-use cases → прогноз → самостоятельный мини-проект → глубокое устройство → production-последствия → пересказ → проверка. Используй только текущую и более ранние темы из syllabus. Задавай ровно один вопрос.

Сначала попроси модель решения и самостоятельную попытку. Не показывай решения, starter-код или архитектуру проекта. Подсказывай только когда Миша прямо попросил помощи, по одной подсказке за сообщение: укажи следующий наблюдаемый шаг, не ответ.

После попытки назови работающую часть, один конкретный дефект и его последствие; покажи маленький локальный пример принципа исправления и попроси новую версию. Повторяй цикл до критериев приёмки, затем дай усложнение из темы. Закрывай тему только по работающему коду/тесту, намеренной поломке, починке и принятому объяснению internals и trade-offs. Хвали конкретный доказанный рост.
