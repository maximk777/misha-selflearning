# Доказательная готовность

Готовность не равна числу прочитанных файлов. Она подтверждается материалом в `progress/`: выводом команды, проходящими тестами, работающим кодом или устным объяснением, принятым экзаменатором. Этот документ задаёт общую шкалу; вопросы и способ выставления баллов — в [RUBRIC.md](../exams/RUBRIC.md).

## Пять уровней владения

| Уровень | Что наблюдаем | Минимальное доказательство | Чего ещё недостаточно |
|---|---|---|---|
| не изучено | Нет проверенного контакта с темой | Нет доказательства | «Я когда-то видел» |
| узнаёт | Узнаёт термин или готовый пример | Верно объясняет назначение в чужом коде после подсказки | Повторить определение без запуска |
| объясняет | Своими словами объясняет причину и trade-off | Устный пересказ плюс один follow-up без смены модели | Рабочий код, который не может объяснить |
| применяет | Сам выбирает и запускает решение для известного сценария | Тест/команда/SQL/код; умеет намеренно сломать и починить | Случайно полученный результат без диагностики |
| защищает решение | Сравнивает варианты, называет ограничения и проверяет риск | Практическая задача, защита trade-off, follow-up и доказательство | «Так принято» без условий применимости |

Если ответ слабый, его не заменяет подсказка наставника: фиксируется верная модель, затем даётся маленькая проверка на 10–15 минут. После верной самостоятельной попытки можно повысить уровень.

## Ворота готовности к собеседованию

| Блок | Минимальная готовность | Ворота «готов обсуждать как middle» | Доказательство |
|---|---|---|---|
| Go | `применяет` базовый Go | Объясняет values/pointers, interfaces, errors, tests; защищает решение API без лишних абстракций | Проходящие Go tests, чтение кода, mini mock |
| Конкурентность | `применяет` context и sync | Находит race/deadlock/leak, выбирает channel/mutex/atomic, завершает workers | `go test -race`, разбор дампа или контролируемого сбоя |
| HTTP | `применяет` handler и client | Защищает timeouts, cancellation, validation/errors, shutdown и тесты | Тестируемый сервис и `httptest` |
| PostgreSQL | `применяет` SQL/transactions | Объясняет MVCC/locks/index choice, диагностирует plan/deadlock, называет последствия long tx | SQL-сценарий, `EXPLAIN (ANALYZE, BUFFERS)`, две сессии locks |
| Messaging | `объясняет` Redis/Kafka semantics | Защищает cache invalidation, idempotency, retries/DLQ и ordering; не обещает невозможный exactly-once | Сценарий повтора/потери и выбранный ключ идемпотентности |
| Архитектура | `объясняет` boundaries | Сравнивает monolith/microservices, outbox/CDC/saga, связывает CAP с конкретным отказом | Защита схемы и two trade-offs |
| Инфраструктура | `объясняет` Compose/Nginx/Kubernetes | Объясняет health/readiness, limits, LB/timeouts/retry storm, безопасно читает manifest | Запуск реплик через Nginx и устный разбор manifest |

## Правило итоговой готовности

К контрольной дате «готов к тренировочным собеседованиям на Go middle» можно поставить только когда все семь ворот достигли хотя бы уровня `объясняет`, а Go, конкурентность, HTTP и PostgreSQL — `применяет`; пройдены четыре mock-набора и у каждого критичного пробела есть повторная проверка. Формулировка не обещает оффер: это честная учебная готовность по накопленным доказательствам.
