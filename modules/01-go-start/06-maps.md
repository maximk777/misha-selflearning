## Модель

Чтение из nil map безопасно и возвращает zero value типа значения. Запись в nil map паникует: для записи нужна `make(map[K]V)` или literal. Форма `value, ok := m[key]` отделяет «нет ключа» от «есть zero value».

## Где это применяется в реальном backend

1. **Индекс заказов по ID** — map ускоряет lookup; duplicate ID требует явной политики, а не молчаливой перезаписи.
2. **Агрегация по статусу** — отсутствующий счётчик удобно начинается с zero value; отсутствие и сохранённый ноль различают через `ok`.
3. **Конфигурация правил** — чтение nil map безопасно, запись panic; map должен создать владелец mutation.

## Глубокое погружение

Map — reference-like descriptor runtime-таблицы; присваивание не копирует все entries. Порядок `range` не гарантирован, адрес элемента map взять нельзя, concurrent запись без синхронизации недопустима (это изучается позже). Ключ должен быть comparable. Удаление отсутствующего ключа безопасно. Инварианты — уникальность ID и явное различие missing/zero. Докажи отдельными простыми `go test` cases или запусками для duplicate/missing/zero и не сравнивай строковый вывод с единственным порядком `range`.

### Критерии приёмки

- [ ] запись выполняется только в созданную map;
- [ ] missing не смешан с zero value;
- [ ] отчёт не зависит от порядка `range map`.
