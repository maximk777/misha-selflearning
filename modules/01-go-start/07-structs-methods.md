## Модель

`struct` объединяет данные в именованный тип. Метод с receiver делает операцию частью поведения типа. Pointer receiver выбирают, когда метод должен изменить исходное значение или когда копирование дорого; value receiver — для маленького значения без изменения его состояния.

## Где это применяется в реальном backend

1. **Модель заказа** — struct группирует ID, сумму и статус; публичные поля не должны позволять обойти инварианты без причины.
2. **Переход статуса** — method выражает допустимую операцию; pointer receiver нужен для изменения исходного заказа.
3. **Формирование строки отчёта** — value receiver читает небольшое значение; копирование большого struct может стать лишней стоимостью.

## Глубокое погружение

Receiver — обычный первый параметр с method syntax. Value receiver получает копию; pointer receiver может менять объект и расширяет method set `*T`. Zero value типа должен быть либо полезен, либо явно отклонён. Ownership mutation остаётся у caller, передавшего pointer. Edge cases: nil pointer receiver, копирование struct с reference-like полями, несогласованные методы переходов. Докажи простыми тестами состояния до/после и компиляционной проверкой method call.
