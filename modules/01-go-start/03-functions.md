## Результат за 5–15 минут

Добавь к starter функцию, которая принимает имя и номер попытки и возвращает строку; вызови её из `main` и запусти `go run .`.

## Модель

Параметры и результаты имеют типы в сигнатуре. Go часто возвращает значение вместе с `error`; ошибка идёт последней, чтобы обычный результат читался первым. Функция не ловит ошибку «сама» — caller выбирает дальнейшее действие.

## Прогноз → опыт → поломка

**Один вопрос:** скомпилируется ли функция, в которой один return branch возвращает одно значение, а другой — два? Проверь в отдельном временном фрагменте, прочитай ошибку компилятора и удали фрагмент.

## Пересказ и сдача

**Один вопрос:** зачем `error` обычно последний результат? Покажи рабочую функцию и объясни, кто отвечает за обработку ошибки.

## Где это применяется в реальном backend

1. **Валидация заказа** — функция возвращает нормализованную сумму и `error`; caller решает, отклонить запись или остановить batch.
2. **Расчёт комиссии** — чистая функция изолирует бизнес-правило; скрытая mutation входа делает повторный расчёт непредсказуемым.
3. **Формирование отчёта** — маленькие функции отделяют подсчёт от вывода; слишком широкая сигнатура размывает ownership.

## Глубокое погружение

Аргументы передаются по значению, сигнатура является контрактом compile time. Несколько результатов возвращаются как согласованный набор; каждый путь должен вернуть нужные типы. Инвариант: ошибка не сопровождается результатом, который caller может принять за успешный без договорённости. Слишком широкая функция усложняет чтение и повторное использование. Докажи контракт запусками с корректным и ошибочным входом и компиляционной поломкой сигнатуры.

## Мини-проект

### Результат

Расширь `order-report`: вынеси проверку суммы, расчёт комиссии и форматирование строки заказа в отдельные функции.

### Разрешённые знания

Все темы Go Start до functions: синтаксис, packages/modules, функции, несколько return и `error`.

### Проверка

Из корня: `cd project/order-report && go test ./... && go run .`; tests покрывают валидную, нулевую и отрицательную сумму.

### Критерии приёмки

- [ ] caller явно обрабатывает каждый `error`;
- [ ] функции имеют одну объяснимую ответственность;
- [ ] ученик защищает порядок результатов и границу валидации.

### Усложнение после первой версии

Добавить функцию скидки с отдельным признаком, была ли скидка применена.
