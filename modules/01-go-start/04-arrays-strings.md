## Модель

`[N]T` — array фиксированной длины; длина входит в его тип. `string` хранит неизменяемые bytes, обычно UTF-8. `range` по string декодирует runes, поэтому `len(text)` и число символов могут различаться.

## Пересказ и сдача

**Один вопрос:** почему индекс `string` возвращает byte? Покажи зелёный test и объясни, когда нужны bytes, а когда runes.

## Где это применяется в реальном backend

1. **Проверка кода заказа** — bytes подходят для ASCII-префикса, runes — для пользовательского текста; индекс byte может разрезать UTF-8 символ.
2. **Ограничение заголовка отчёта** — лимит по символам требует runes, лимит протокола обычно задаётся bytes; смешение лимитов ломает данные.
3. **Фиксированная контрольная сумма** — array выражает точный размер; `[16]byte` и `[32]byte` являются разными типами.

## Глубокое погружение

Array копирует все элементы при присваивании. String — неизменяемая последовательность bytes; `len` считает bytes, `range` декодирует UTF-8 и может выдать `RuneError` для повреждённых данных. Конвертация `string`/`[]byte` обычно копирует память. Инвариант — не обрезать строку посередине encoding. Докажи таблицей ASCII/кириллица/emoji/невалидный UTF-8 и тестами byte length против rune count.
