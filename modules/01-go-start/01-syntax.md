# Синтаксис и первый запуск

> Уровень: `start`
> Время: 45 минут
> Предпосылки: Go установлен
> Практика: `labs/go/01-syntax/LAB.md`
> Экзамен: `exams/go-start.md`, GS-1

## Результат за 5–15 минут

Выполни `cd labs/go/01-syntax/starter && go run .`: программа дважды напечатает приветствие с номером запуска.

## Модель

Go компилирует исходный код заранее. Переменная получает конкретный тип, а блоки `if` и `for` задают обычный поток выполнения. Короткая запись `:=` доступна только там, где появляется хотя бы новая переменная.

## Прогноз → опыт → поломка

**Один вопрос:** сколько строк будет в выводе и почему? Запусти программу. Затем замени `attempt <= 2` на `attempt < 2`, зафиксируй изменение и верни исходное поведение.

## Пересказ и сдача

**Один вопрос:** что делает условие цикла до каждой итерации? Покажи вывод `go run .`, поломку и починку; затем ответь на GS-1.

## Где это применяется в реальном backend

1. **Проверка заказа** — условия отклоняют отрицательную сумму, цикл обходит позиции; без точной границы цикла позицию легко пропустить или обработать дважды.
2. **Пакетный отчёт** — переменные накапливают количество и сумму заказов; локальный счётчик нельзя принимать за устойчивое хранилище между запусками.
3. **Повтор операции** — цикл задаёт ограниченное число попыток; бесконечный retry без паузы и лимита перегружает зависимость.

## Глубокое погружение

Тип переменной известен компилятору, область видимости задаётся блоком, а zero value появляется без явной инициализации. Инвариант CLI: каждый принятый заказ учитывается один раз. Ошибки границ `for`, shadowing через `:=` и integer overflow дают неверный отчёт. Докажи поведение таблицей входов и выводом `go run .`; компилятор доказывает типы, но не бизнес-инвариант.

## Мини-проект

### Результат

Открой предоставленный пустой scaffold `project/order-report`: module уже настроен, создавать `go.mod` не нужно. В `main.go` по встроенному набору сумм напечатай число принятых и отклонённых заказов и общую сумму принятых.

### Разрешённые знания

Только переменные, базовые типы, `if`, `for`, `fmt` и запуск Go-программы.

### Проверка

Из корня: `cd project/order-report && go run .`; затем вручную меняй встроенный набор для пустого, нулевого и отрицательного входа.

### Критерии приёмки

- [ ] каждый вход попадает ровно в одну категорию;
- [ ] результат пересчитывается после намеренного изменения границы условия;
- [ ] ученик объясняет типы, zero value и условие остановки цикла.

### Усложнение после первой версии

Добавить отдельный счётчик заказов дороже заданного порога.
