# Wrapping ошибок

> Уровень: `interview`
> Время: 60 минут
> Предпосылки: errors
> Практика: `labs/go/06-errors-panic/LAB.md`
> Экзамен: `exams/go-core.md`, GC-3

## Результат за 5–15 минут

Запусти `go -C labs/go/06-errors-panic/starter test ./...`. `errors.Is` находит `ErrNotFound` внутри контекстного сообщения.

## Модель

Контекст объясняет операцию, а `%w` сохраняет цепочку причин. `errors.Is` отвечает на вопрос «эта ли категория ошибки?», `errors.As` извлекает подходящий конкретный тип. Не сравнивай текст ошибки для бизнес-ветвления.

## Прогноз → опыт → поломка

**Один вопрос:** почему `%v` выглядит похоже, но ломает `errors.Is`? Проверь заменой в lab, затем почини `%w`.

## Пересказ и сдача

**Один вопрос:** какую информацию видит пользователь, а какую лог? Покажи test и ответь на GC-3.

## Где это применяется в реальном backend

1. **Не найден заказ** — sentinel даёт стабильную категорию, wrapping добавляет operation/ID; сравнение текста ломается при смене сообщения.
2. **Ошибка доменного правила** — typed error несёт поля для решения caller; наружу нельзя утекать чувствительным контекстом.
3. **Batch validation** — caller различает повторяемую и окончательную ошибку; indiscriminate retry усиливает сбой.

## Глубокое погружение

`%w` строит unwrap chain, `errors.Is` обходит её по identity/custom `Is`, `errors.As` назначает первый совместимый тип. Инвариант — каждый слой добавляет контекст один раз и сохраняет причину. Edge cases: несколько wrapped причин, nil error, typed nil внутри error interface, потеря chain через `%v`. Диагностический текст растёт с каждым слоем. Докажи несколькими отдельными tests для Is/As и проверкой публичного сообщения через `go -C project/order-report test ./...`.

Продолжи `project/order-report/domain`: добавь категории ошибок для неизвестного заказа, недопустимого перехода и неверной суммы, сохранив контекст операции.

### Разрешённые знания

Go Start, interfaces, error wrapping, `errors.Is` и `errors.As`.

### Проверка

Из корня репозитория: `go -C project/order-report test ./...` на каждую категорию, wrapped cause и неожиданный error.

### Критерии приёмки

- [ ] caller ветвится по категории/типу, не тексту;
- [ ] причина не теряется;
- [ ] ученик отделяет user message от diagnostic context.

### Усложнение после первой версии
